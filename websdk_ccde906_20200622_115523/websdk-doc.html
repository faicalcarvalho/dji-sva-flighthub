1. Overview

1.1. About the REST API

The FlightHub WebSDK REST API v3 document offers complete control over drone, team, pilot user,
flight statistics and live streaming.

1.2. Version information

Version : 3.0.0

1.3. URI scheme

Host : Server_Address
BasePath : /fhsdk/v3
Schemes : HTTP, HTTPS

1.4. Tags

  • User : User Management

  • Team : Team Management

  • Drone : Drone Management

  • Flight Records : Flight Records

  • Flight Statistics : Flight Statistics

  • Live Streaming : Live streaming

  • Real-time Report: Real-time report

1.5. Consumes

  • application/json

1.6. How to use

There are two kinds of user using the REST API:
1. Application Server that controls FlightHub directly with the superuser permission. Application Server must be authenticated with two keys : AppId and SignKey.
2. End User normally means APP(likes DJI Pilot) that controls the drone directly and connects to FlightHub. End User must login using account and password and should be authenticated with Common AppId(flighthub-pilot by default) and dynamic generated SignKey for each session.

2. Security

2.1. Overview

Headers(FH-AppId, FH-ReqId, FH-Ts, FH-Sign) should be added to HTTP request.
FH-Token header should be added if connected as End User.

2.1.1. How to sign a request

Step 1. Derive the master signing key

Pseudo code for deriving the master signing key is as below:

SignKey1 = HMAC_SHA1(FH-Ts, SignKey)
MasterSignKey = HMAC_SHA1(FH-AppId, SignKey1)

HMAC_SHA1 represents an HMAC-SHA1 function that returns output in binary format. The first parameter of the function is data, and the second parameter is the key.
To generate the SignKey1, you should use SignKey as the hash key, FH-Ts as the data.
To generate the master signing key, you should use SignKey1 as the hash key, FH-AppId as the data.

Step 2. Calculate the signature.

Sign = std_base64(HMAC_SHA1(sign_data, MasterSignKey))

Data to sign (sign_data) is the content of the whole request body.
For GET request, sign_data is empty string.

Use the master signing key as key to calculate sign of the sign_data.
The final signature string is the base64 encoded(standard encoding with padding) string of the sign.

Step 3. Add the signature to HTTP request
You can put the signature string in one of the two ways:

  • the HTTP header named FH-Sign

  • query string named FH-Sign.

2.1.2. Sample code

fhTs  := 1543478800840       // the time in milliseconds since the UNIX epoch
appID := "flighthub-admin"
reqID := "reqXXXX"           // unique ID for each request, for tracing purpose.

req.Header.Set("FH-AppId", appID)
req.Header.Set("FH-Ts",fhTs)
req.Header.Set("FH-ReqId", reqID)

key1 := HmacSha1([]byte(fhTs), []byte(key)) // HmacSha1 result is in binary format, not hex or base64 encoded.
masterKey := HmacSha1([]byte(appID), key1)

var buf []byte
if req.Body != nil {
        buf, _ = ioutil.ReadAll(req.Body)
}
rc := ioutil.NopCloser(bytes.NewBuffer(buf))
req.Body = rc

req.Header.Add("FH-Sign", Sha1HashBase64(buf, masterKey)) // use base64 encoded hash when put in FH-Sign Header
